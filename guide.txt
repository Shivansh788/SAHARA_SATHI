SAHARA Saathi - Agent-Friendly Setup Guide
==========================================

Purpose
-------
This runbook is designed so another engineer/agent can set up and verify the project quickly with deterministic commands.

Quick Start (Single Line)
-------------------------
Run this from project root:

chmod +x bootstrap.sh run.sh && ./bootstrap.sh --run

Open:
http://localhost:8501


Deterministic Setup Steps
-------------------------
1) One-time setup
    chmod +x bootstrap.sh run.sh
    ./bootstrap.sh

2) Start app
    ./run.sh

3) Health check
    curl -s http://127.0.0.1:8501/api/health

4) Stop app
    Ctrl+C in the running terminal


If setup is already done and you only want to rerun migration:
    source venv/bin/activate
    python db_migrate.py


Configuration Rules
-------------------
- Use config.py as single source of truth for app settings.
- Use .env for local sharing and env override in production.
- Core runtime values:
  - APP_HOST / APP_PORT
  - COEAI_API_KEY
    - GEMINI_API_KEY
    - OPENAI_API_KEY
  - SAHARA_TOKEN_SECRET
  - DATA_GOV_* values

.env bootstrap:
    cp .env.example .env
    # edit .env and set only required keys


Post-Setup Smoke Tests
----------------------
1) Ask endpoint
    curl -sS -X POST http://127.0.0.1:8501/api/ask -H "Content-Type: application/json" -d '{"query":"pm kisan eligibility","session_id":"smoke1","language":"english","mode":"citizen"}'

2) Events endpoint
    curl -s http://127.0.0.1:8501/api/events

3) NGOs endpoint
    curl -s "http://127.0.0.1:8501/api/ngos?location=dehradun"


Known Behavior
--------------
- Chat profile memory persists only explicit user-declared facts.
- NGO events are loaded from data/ngo_events.csv into SQLite events.
- If Gemini/COEAI are unavailable, pipeline falls back safely.


Troubleshooting
---------------
Port busy:
  pkill -f "python.*app.py"
    ./run.sh

Import/module errors:
  source venv/bin/activate
  pip install -r requirements.txt

Broken virtual environment (clean rebuild):
    rm -rf venv
    ./bootstrap.sh

Rebuild DB data:
  python db_migrate.py
