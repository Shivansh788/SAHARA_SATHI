SAHARA SAATHI - TILL NOW STATUS
Date: 2026-03-27

This file summarizes everything implemented in this upgrade pass in a human-friendly way.

FREE-FIRST UPDATE (latest)
- LLM pipeline is now free-first: local Ollama is primary generation path.
- Optional external paid provider (COEAI) is only fallback when explicitly configured.
- Main app no longer blocks when paid key is missing.
- Added optional offline TTS support using pyttsx3; fallback to gTTS remains.
- Removed mandatory coeai dependency from requirements.

SMALL FIX BATCH (latest)
- Frontend login/signup now calls backend auth APIs instead of only local UI simulation.
- Chat mode selector added in UI (Citizen/Worker/Explorer) and mode is sent to /api/ask.
- Added /api/health endpoint for quick runtime checks.
- Org register endpoint no longer exposes OTP by default; OTP hint is only shown when SAHARA_DEV_OTP_HINT=true.

SMALL FIX BATCH 2 (latest)
- Added profile data binding from authenticated user details (name/identifier/state/category).
- Added in-app toast notifications and replaced blocking alert popups for auth and UX messages.
- Added scheme bookmark save action from scheme cards and profile-side bookmark listing/removal UI.
- Added OTP expiry + attempt tracking in persistence layer.
- Added org OTP resend endpoint: POST /api/org/resend-otp.
- Org verification now returns detailed reason payload (invalid/expired/attempts/suspended).
- Startup now warns when SAHARA_TOKEN_SECRET is left at default.

==================================================
1) Retrieval Quality Upgrades (Implemented)
==================================================
- Converted raw RAG documents into structured records with metadata.
  Includes source type, source name, category, eligibility, and searchable text.
- Added versioned RAG cache handling in rag_data.pkl.
- Updated embedding preparation to use clean text fields.
- Implemented proper Reciprocal Rank Fusion (RRF) with configurable scoring.
  Flow now supports larger candidate set (top-50) before final selection.
- Implemented cross-encoder reranking support.
  Uses: cross-encoder/ms-marco-MiniLM-L-6-v2 when available.
  Falls back safely to lexical heuristic if model loading fails.
